# Pattern: Constraint stack

Order matters:

1. **Goal** (one primary outcome)  
2. **Hard constraints** (must / must not)  
3. **Soft preferences** (nice to have)  
4. **Format** (how to respond)  
5. **Done checks**

Hard constraints beat preferences when they conflict. Say so explicitly.
